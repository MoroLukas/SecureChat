TC-002 – Login utente registrato

Riferimento: TC-002
REQ-002 – Nome: Login utente

Descrizione:
Verificare che un utente registrato possa effettuare il login.

Prerequisiti:

Server Node.js in esecuzione

Utente esistente nella tabella users

Accesso alla pagina “Login”

Procedura:

Accedere alla pagina “Login”

Inserire:

Username: testuser01

Password: Password123!

Cliccare sul pulsante “Login”

Risultati attesi:

Login effettuato con successo

Redirect alla dashboard / chat page

Creazione sessione o token JWT valido

TC-003 – Login con credenziali errate

Riferimento: TC-003
REQ-003 – Nome: Login fallito

Descrizione:
Verificare che il sistema rifiuti credenziali errate.

Prerequisiti:

Server Node.js in esecuzione

Utente esistente

Accesso alla pagina “Login”

Procedura:

Accedere alla pagina “Login”

Inserire:

Username: testuser01

Password: WrongPassword

Cliccare su “Login”

Risultati attesi:

Messaggio di errore “Credenziali non valide”

Nessuna sessione/token creato

Nessun accesso alla chat

TC-004 – Invio messaggio tra due utenti

Riferimento: TC-004
REQ-004 – Nome: Invio messaggio

Descrizione:
Verificare che un utente possa inviare un messaggio a un altro utente.

Prerequisiti:

Server Node.js in esecuzione

Utenti userA e userB esistenti

Utente userA loggato

Tabella messages vuota

Procedura:

Effettuare il login come userA

Aprire la chat con userB

Inserire il testo: “Ciao, questo è un messaggio di test”

Cliccare su “Send”

Risultati attesi:

Messaggio visualizzato nella chat

Record inserito nella tabella messages

Stato messaggio = “sent”

TC-005 – Verifica persistenza messaggi nel DB

Riferimento: TC-005
REQ-005 – Nome: Persistenza messaggi

Descrizione:
Verificare che i messaggi inviati siano correttamente salvati nel database.

Prerequisiti:

Messaggio inviato con successo (TC-004)

Accesso al DB

Procedura:

Eseguire la query SQL:

SELECT sender_id, receiver_id, content, created_at 
FROM messages 
WHERE content='Ciao, questo è un messaggio di test';


Risultati attesi:

Il messaggio è presente nel DB

I dati mittente e destinatario sono corretti

Timestamp valorizzato

TC-006 – Visibilità messaggi: DB vs GUI

Riferimento: TC-006
REQ-006 – Nome: Visualizzazione messaggi

Descrizione:
Verificare che i messaggi salvati nel DB siano correttamente visualizzati nella GUI.

Prerequisiti:

Messaggi presenti nella tabella messages

Utente userB loggato

Procedura:

Effettuare login come userB

Aprire la chat con userA

Confrontare i messaggi visualizzati con i record DB

Risultati attesi:

Tutti i messaggi presenti nel DB sono visibili nella GUI

Nessun messaggio duplicato

Ordine cronologico corretto

TC-007 – Sicurezza: messaggi non visibili a utenti non autorizzati

Riferimento: TC-007
REQ-007 – Nome: Sicurezza messaggi

Descrizione:
Verificare che un utente non possa visualizzare messaggi di altre conversazioni.

Prerequisiti:

Esiste una conversazione tra userA e userB

Utente userC esistente e loggato

Procedura:

Effettuare login come userC

Tentare di accedere alla chat tra userA e userB via URL diretto o API

Risultati attesi:

Accesso negato (403 / Unauthorized)

Nessun messaggio visualizzato

Nessun dato sensibile esposto
